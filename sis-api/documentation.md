## Overview

There are **two FastAPI services** in the stack:

| Service          | Port (dev) | Purpose                                                     |
| ---------------- | ---------- | ----------------------------------------------------------- |
| `sis-api`        | 8002       | Admin operations + data read endpoints (this document)      |
| `sis-api-glosis` | 8006       | Read-only GloSIS Federation endpoints (manifest / profile / observation) — opt-in |

In production both are reached through `sis-nginx` on port 80/443 (`/api/...` and `/glosis/...` respectively); the direct ports are dev conveniences.

`/docs`, `/redoc` and `/openapi.json` are **disabled by default**. Set `ENABLE_DOCS=true` in the env to re-enable them for local development.

## Authentication

Three auth mechanisms, used by different endpoint classes:

| Mechanism           | Header                          | Used for                                                   |
| ------------------- | ------------------------------- | ---------------------------------------------------------- |
| **JWT (user)**      | `Authorization: Bearer <token>` | All admin + most operational endpoints in `sis-api`        |
| **API key**         | `X-API-Key: <key>`              | Public read endpoints (`/api/manifest`, `/api/profile`, …) |
| **Federation token**| `X-API-Key: <key>`              | `sis-api-glosis` endpoints only — see [GloSIS](#glosis-federation) |

### JWT login

```bash
curl -X POST http://localhost:8002/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"user_id":"admin","password":"<password>"}'
```

`user_id` is a plain string (any non-empty value). The first admin is `admin`; its password is generated by `deploy.sh` and printed once when the deploy finishes.

Response:
```json
{ "access_token": "eyJhbGciOiJIUzI1NiIs…", "token_type": "bearer" }
```

Tokens expire after `ACCESS_TOKEN_EXPIRE_MINUTES` (default **60 minutes**), and are **revoked** when the user changes their password (via the `iat` claim + `password_changed_at` column). Deactivated users (`is_active = false`) are rejected on every request.

### Account lockout

After **5 consecutive failed logins** the account is locked for **15 minutes**. The lockout response is indistinguishable from "wrong password" (same generic error) so it doesn't leak state. A successful login resets the counter.

### Authorization model

| Section                                                | Required auth          | Notes                                          |
| ------------------------------------------------------ | ---------------------- | ---------------------------------------------- |
| Administration tab (users, API clients, settings, GloSIS) | **JWT (admin)**     | `is_admin = true` enforced server-side         |
| ETL Delete dataset (`DELETE /api/etl/datasets/{table}`)| **JWT (admin)**        | Destructive — drops the staging table          |
| Everything else mutating (layers, ETL upload/ingest/prune, codelist creates, soil-profile publish/limit/blur, sync)  | **JWT (any user)**     | Login required                                  |
| Read endpoints under `/api/*`                          | **API key**            | Header `X-API-Key`                              |
| `/health`, `/`                                         | **Public**             |                                                |

CORS origins are restricted to the operator-configured frontend domain (`localhost`-only in dev). The federation API (`sis-api-glosis`) uses `*` since it's API-key-protected.

## Endpoints

### Authentication endpoints

| Method | Path                  | Auth           | Description                                  |
| ------ | --------------------- | -------------- | -------------------------------------------- |
| POST   | `/api/auth/login`     | Public         | Login → returns JWT                          |
| GET    | `/api/auth/verify`    | JWT            | Verify token validity, returns user record   |
| PATCH  | `/api/users/me`       | JWT            | Self-update (change own user_id or password) |

`PATCH /api/users/me` body — supply the current password and either or both new values:
```json
{ "current_password": "…", "new_user_id": "alice", "new_password": "…" }
```
If `new_user_id` changes, the response includes a fresh `access_token` so the SPA can continue seamlessly.

### ETL pipeline

CSV-based bulk ingest of soil data. All steps below require login but **not admin**, except `DELETE` (admin-only).

| Method | Path                                                  | Description                                                |
| ------ | ----------------------------------------------------- | ---------------------------------------------------------- |
| PUT    | `/api/etl/metadata`                                   | Save the dataset's project + authors                       |
| GET    | `/api/etl/project/{project_id}/authors`               | List authors linked to a project                            |
| POST   | `/api/etl/upload`                                     | Upload a CSV — creates a staging table in `soil_data_upload` |
| GET    | `/api/etl/datasets`                                   | List the operator's uploads                                 |
| GET    | `/api/etl/datasets/{table_name}/preview`              | First 20 rows of the staging table                          |
| GET    | `/api/etl/datasets/{table_name}/columns`              | Saved column mappings + per-column validation              |
| PUT    | `/api/etl/datasets/{table_name}/columns`              | Save column mappings, source EPSG, dataset's project_id    |
| POST   | `/api/etl/datasets/{table_name}/validate`             | Run the validation rules (see below). Returns structured per-column results, country-bounds report, and missing required destinations |
| POST   | `/api/etl/datasets/{table_name}/ingest`               | Push validated rows into `soil_data.{plot,profile,element,specimen,result_num}` |
| PATCH  | `/api/etl/datasets/{table_name}/cells`                | Inline-edit cell values in the staging table               |
| POST   | `/api/etl/datasets/{table_name}/prune`                | Reverse ingest — delete the rows this CSV created          |
| DELETE | `/api/etl/datasets/{table_name}`                      | **Admin only** — drop the staging table and metadata rows  |

#### CSV upload caps

- **50 MB** byte limit
- **200 000** data-row limit
- Staging table name capped at 40 chars (avoids 63-char Postgres-identifier collisions)
- All staging tables live in schema `soil_data_upload`; columns are `TEXT` and column names are quoted via `psycopg2.sql.Identifier`

#### Plot-CSV provenance

Every plot inserted by an ETL ingest is tagged with `soil_data.plot.csv = <staging_table_name>`. This makes **prune** dataset-scoped instead of project-scoped — pruning one CSV doesn't wipe other CSVs sharing the same project.

#### Auto-derived columns at ingest

The user maps CSV columns to a single combined destination dropdown (label → table.column). Two destinations are **auto-derived** rather than user-mapped:

- `soil_data.site.site_id` is set equal to the dataset's `project_id` (reused across CSVs of the same project, never created separately).
- `soil_data.profile.profile_code` is set equal to the value mapped to "Profile code" (which writes to `plot.plot_code`).

#### Unit conversion at ingest

Result values are stored in the **canonical unit** for the (property, procedure) — i.e. `observation_num.unit_of_measure_id`. The mapped source unit (`uploaded_dataset_column.unit_of_measure_id`) plus `soil_data.unit_conversion` give the multiplier; if source = canonical, no conversion. The same conversion runs in the validator before bounds-checking, so the popup shows correct out-of-bounds row numbers.

### GloSIS Federation

Admin-only configuration in `sis-api`; the federation data itself lives on `sis-api-glosis` (port 8006 in dev, `/glosis/*` via nginx in prod).

#### Admin endpoints (sis-api)

| Method | Path                                | Description                                                     |
| ------ | ----------------------------------- | --------------------------------------------------------------- |
| GET    | `/api/glosis/status`                | `{ enabled, token }` — token's plaintext key included so the admin UI can show it for copy-paste to the Discovery Hub |
| POST   | `/api/glosis/enable`                | Set `GLOSIS_FEDERATION_ENABLED=true`; mint a singleton token if none exists, reactivate if inactive |
| POST   | `/api/glosis/disable`               | Set the setting to false — token row kept so re-enabling reuses it |
| POST   | `/api/glosis/disable_and_delete`    | Disable AND delete the token (audit rows referencing it have their `api_client_id` nulled) |

#### Data endpoints (sis-api-glosis)

Each requires `X-API-Key: <federation token>` AND `GLOSIS_FEDERATION_ENABLED='true'`. Connects to Postgres as the read-only `sis_glosis` role — SELECT only on the three federation views + the federation token view + `api.setting`, plus INSERT on `api.audit`.

| Method | Path           | Description                                       |
| ------ | -------------- | ------------------------------------------------- |
| GET    | `/manifest`    | Soil property manifest (`vw_api_manifest`)        |
| GET    | `/profile`     | Profiles (`vw_api_profile`, same filters as the SPA — `is_published`, `profile_limit`, blur) |
| GET    | `/observation` | Observations (`vw_api_observation`)               |

The exposed data is **exactly** what's visible on this SIS — controlled via Layers → Soil profiles.

## Validation rules

Run by `POST /api/etl/datasets/{table_name}/validate`. Per CSV column:

| Destination                        | Rule                                                                 |
| ---------------------------------- | -------------------------------------------------------------------- |
| `plot.plot_code`                   | Text. Rows sharing the same value are merged into one plot/profile (must agree on Longitude and Latitude) |
| `plot.type`                        | Enum: `TrialPit`, `Borehole`                                        |
| `plot.altitude`                    | Smallint                                                            |
| `plot.positional_accuracy`         | Smallint                                                            |
| `plot.sampling_date`               | Date `YYYY-MM-DD`                                                   |
| `plot.geom (longitude)`            | Real, `[-180, 180]`                                                 |
| `plot.geom (latitude)`             | Real, `[-90, 90]`                                                   |
| `element.upper_depth`              | Whole number, `[0, 1000]`                                           |
| `element.lower_depth`              | Whole number ≥ 0                                                    |
| `element.type`                     | Enum: `Horizon`, `Layer`                                            |
| `element.horizon`                  | Free text (no format check)                                          |
| `result_num.value`                 | Number, converted to canonical unit, then compared against `observation_num.value_min/value_max` |

**Cross-row consistency** (also run):
- `upper_depth < lower_depth` for each element row.
- Layers per profile must be **contiguous**: sorted by upper, each layer's lower equals the next layer's upper.
- Rows sharing the same `profile_code` must agree on Longitude and Latitude.
- **Country bounds**: at least **95%** of rows' (lon, lat) points must fall inside `spatial_metadata.country.geom_convexhull` for the configured `COUNTRY_CODE`.

**Required destinations** (the column must be mapped at least once): Profile code, Longitude, Latitude, Sampling date, Upper depth, Lower depth, Soil property.

**Defence in depth.** The application validation above is a UX layer. A DB trigger (`soil_data.trg_check_result_value`) re-checks each inserted/updated `result_num.value` against `observation_num.value_min/value_max`. Bypassing the validator does not bypass the trigger.

## Database schema

Major tables touched by the API. Full schema lives in `sis-database/sis-database_latest_with_codelist.sql`.

### `api.user`

| Column                  | Type        | Purpose                                              |
| ----------------------- | ----------- | ---------------------------------------------------- |
| `user_id`               | text PK     | Login string (any non-empty value)                  |
| `password_hash`         | text        | bcrypt                                              |
| `is_active`             | bool        |                                                      |
| `is_admin`              | bool        | Admin gate for Administration endpoints              |
| `created_at` / `updated_at` / `last_login` | timestamptz |                              |
| `password_changed_at`   | timestamptz | JWTs issued before this are rejected                |
| `failed_login_attempts` | int         | Reset on success                                    |
| `locked_until`          | timestamptz | Lockout cutoff after 5 failures                     |

### `api.api_client`

| Column            | Purpose                                              |
| ----------------- | ---------------------------------------------------- |
| `api_client_id`   | Identifier                                            |
| `api_key`         | Plaintext key (matched directly)                     |
| `description`     | Free text; `'glosis-federation'` marks federation tokens |
| `is_active`, `expires_at`, `last_login`, `created_at` | Lifecycle fields                            |

Restricted view `api.vw_glosis_federation_token` filters to `description = 'glosis-federation'` — what the read-only `sis_glosis` role can see.

### `api.audit`

Append-only via SQL triggers:
- `BEFORE DELETE` blocks every delete.
- `BEFORE UPDATE` blocks every column change **except** nulling `user_id` / `api_client_id` (needed when a parent row is purged).

| Column          | Purpose                                                            |
| --------------- | ------------------------------------------------------------------ |
| `audit_id`      | PK (identity)                                                       |
| `user_id`       | FK → `api.user`, nullable                                          |
| `api_client_id` | FK → `api.api_client`, nullable                                    |
| `action`        | Free text                                                           |
| `details`       | JSONB                                                               |
| `ip_address`    | `inet`, populated from `X-Forwarded-For` when present              |
| `created_at`    | default `now()`                                                     |

### `api.layer`

Layer metadata + WMS URLs.

| Column                                                       | Notes |
| ------------------------------------------------------------ | ----- |
| `layer_id`, `project_id`, `project_name`, `property_name`    | First two seeded from sync; `project_name` (the Group column) and `property_name` (the Raster name column) are editable inline by any logged-in user |
| `dimension`, `version`, `keywords`                           | Refreshed every sync                                                   |
| `unit_of_measure_id`, `metadata_url`, `download_url`         |                                                                      |
| `get_map_url`, `get_legend_url`, `get_feature_info_url`      |                                                                      |
| `publish` (bool)                                             | Hidden from public listings when false                                |
| `is_default` (bool)                                          | Only one row can be true (enforced at write)                          |

### `api.setting`

Single-table key/value config. The UI hides `DOWNLOAD_BASE_URL` and `GLOSIS_FEDERATION_ENABLED` (infrastructure keys managed elsewhere).

### `api.uploaded_dataset` and `api.uploaded_dataset_column`

Per-CSV provenance for the ETL pipeline. Each upload creates a row in `uploaded_dataset` and one row per CSV column in `uploaded_dataset_column`. The staging table itself lives in schema `soil_data_upload`.

| `api.uploaded_dataset_column` field | Purpose                                          |
| ----------------------------------- | ------------------------------------------------ |
| `destination_table` / `destination_column` | Where this CSV column should write at ingest |
| `property_num_id` / `procedure_num_id` / `unit_of_measure_id` | For `result_num` mappings — source unit (gets converted to canonical) |
| `validation`                        | Last validation status text                       |
| `note`                              | Free text                                         |
| `ignore_column`                     | Set when destination is "(skip)"                  |

### Schemas & views in the data path

- `soil_data.*` — domain tables (project, site, plot, profile, element, specimen, result_num, observation_num, property_num, procedure_num, unit_of_measure, unit_conversion)
- `soil_data_upload.*` — ETL staging tables (one per CSV)
- `spatial_metadata.country` — ISO country codes + names + `geom_centroid` + `geom_convexhull`
- `api.vw_api_manifest` — manifest aggregation, filtered by published profiles
- `api.vw_api_profile` — published profiles (respects `is_published`, `profile_limit`, applies `spatial_blur_m`)
- `api.vw_api_observation` — observations for published profiles
- `api.vw_glosis_federation_token` — restricted view used by `sis_glosis` role

## Security features

(All of these are implemented; the audit checklist + plan lives in `PRODUCTION-PLAN.md`.)

- **Per-deployment secrets** — `deploy.sh` generates `SECRET_KEY`, `WEB_MAPPING_API_KEY`, `POSTGRES_PASSWORD`, `POSTGRES_GLOSIS_PASSWORD` on first run. `.env` is gitignored.
- **Random admin password** — printed once at the end of `deploy.sh`, not stored anywhere else.
- **JWT revocation** — `iat` claim compared to `password_changed_at`; tokens issued before the last password change are rejected.
- **Account lockout** — 5 failures → 15-min lock.
- **Per-account API keys** — federation token stored separately from the web-mapping key; sis-api-glosis can only see federation tokens via the restricted view.
- **Read-only DB role** — `sis_glosis`; the federation service cannot read other API keys or write any table except `api.audit`.
- **Append-only audit** — SQL triggers block UPDATE/DELETE on `api.audit` (except nulling FKs).
- **Admin-only Administration endpoints** — users, API clients, settings, GloSIS controls.
- **CSV upload caps** — 50 MB / 200 000 rows / 40-char base name.
- **XSS-safe metadata popup** — `formatMetadata` escapes every interpolation; link `href` validated to `http(s):` only.
- **Public XSS sink hardening** — admin dashboard `onclick="…('${...}')"` interpolations go through `escapeJsAttr`.
- **CORS allowlist** — no `*` on the JWT-bearing service.
- **`/docs` off by default** — set `ENABLE_DOCS=true` to re-enable.
- **`X-Forwarded-For` trust** — audit IP captured from the proxy header.
- **nginx hardening headers** — `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`, `server_tokens off`.
- **Login rate limit at nginx** — `limit_req` zone on `/api/auth/login` (5 req/s per IP).
- **SSRF guard** — `_validate_pycsw_url` rejects non-http(s) and any host outside an allowlist, and the `requests.get` call disables redirects.
- **Postgres unpublished** — host port 8005 removed from `docker-compose.yml`; use `docker exec` for psql.

Still **open by design** (parked):
- API keys remain **plaintext at rest** — the federation token UX requires the admin be able to re-display it. The web-mapping key is the same column.
- TLS — nginx config has the HTTPS server commented; uncomment + drop cert + uncomment HSTS to enable.

## Error responses

```json
{ "detail": "human-readable message" }
```

| Status | Meaning                                                                              |
| ------ | ------------------------------------------------------------------------------------ |
| 200/201| OK                                                                                   |
| 400    | Validation error in request body                                                     |
| 401    | Missing or invalid JWT / API key, locked account, expired token, etc. (generic)      |
| 403    | Authenticated but lacking privileges (e.g. non-admin hitting an admin endpoint)      |
| 404    | Resource not found                                                                   |
| 413    | CSV upload exceeded size or row caps                                                 |
| 422    | Pydantic schema mismatch                                                              |
| 5xx    | Server error — message generic (DB error strings are not surfaced)                   |
